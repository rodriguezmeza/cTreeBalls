tpcf - 3 Point Correlation Function computation code

SYNOPSIS

       tpcf [ parameter_file_name ] [ options ]

DESCRIPTION

       tpcf - computes the 3 point correlation function using brute force,
       complexity O(N^2).

OPTIONS

       All the options have the structure

       option_name=<option_value>

       except option -help.


       Options and their possible values are:


       -help       By writting


                   tpcf -help


                   you will get the list of all parameters and their default
                   values. An option may have an alias which is a short name of
                   the option. If an option has an alias in the list above it
                   comes after its description surrounded by brackets tagged
                   with 'a:'. For example,


                   option_name=<option_value>    ... Description ... [a: opt]

                   here 'opt' is the short name of the option. In the
                   description of the options below, when an option has an alias
                   it will be noted in the same way but before its description.


       paramfile   is the name file with the values of the input parameters.
                   Overwrite parameters values below. You may input this
                   filename by only writing:

                   tpcf paramfile=parameters_input_file_name

                   Parameter input file may be created by hand with the editor
                   of your choice. Comment lines start with an "%". Follow each
                   name option with a blank space and the option value. The
                   [a: search] is the searching method to use. Default is
                   "tree". Other option is to use brute search method: "direct".
                   When the code is compiled with option OPENMPMACHINE=1, then
                   the best options is "treeomp". When using "direct" option,
                   automatically is run the OpenMP version of it.


       infile      [a: in] will give the name of file to input the analysed
                   data.


       testmodel   [a: tstmodel] is the name of the model of data to create.


       mChebyshev  [a: mcheb] is the total number of chebyshev multipoles to
                   use.


       outfile     [a: o] will give the name of file to save the analysed data.
                   The output is written in column form by default.


       nbody       is the total number of bodies to make as a test case. By
                   default a box of size given by lengthBox is created and there
                   nbody random points are generated. The random generator is
                   initialize with a seed given by seed option.


       seed        gives the seed to init the random generator.


       lengthBox   [a: lbox] is the length of the box side. The box is where the
                   bodies reside.


       sizeHistN   gives the number of bins to create.


       rangeN      is the maximum separtion of bodies to search for sampling.


       statefile   [a: state] is the name of file to save the state of the run.
                   The output is written in binary format. The frequency is
                   given by option stepState.


       stepState   gives the frequency to save the whole state of the run that
                   is save in a file given by statefile in the "root" directory
                   (usually named "Outputs").


       restorefile [a: restore] is the name of file to restore the state of the

EXAMPLES

       time ../tpcf nbody=100000 rangeN=500 sizeHistN=100 lbox=10000 verb=2
       mToPlot=2


       You will have computation written in a file named with option outfile in
       a directory named Output. If this directory does not exist, it will be
       created.  Also will be created a directory named "tmp" where a log file
       will be saved.  You may plot the solutions using nplot2d code:


       nplot2d in=Output/output.txt ws=1 symbolsize=0.1 pj=0 symboltype=4 uc=1:2


       nplot2d is very similar to gnuplot. Then, you can use the one you have
       available.


       Other files can be plotted as:


       nplot2d in=Output/histN.txt,Output_to-compare-with/histN_direct.txt
       uc=1:2,1:2


       nplot2d
       in=Output/mhistZeta.txt,Output/mhistZeta.txt,Output/mhistZeta.txt,Output/mhistZeta.txt,Output/mhistZeta.txt
       uc=1:2,1:3,1:4,1:5,1:6


       Using brute force:


       time tpcf o=output options=compute-HistN verb=2 sizeHistN=40
       sizeHistTheta=40 rangeN=200 nbody=10000 mToPlot=2 lbox=2000
       search=direct-simple-3pcf-omp


       Reading a Takahasi simulation data file:


       time tpcf infile=Takahasi/allskymap_nres12r000.zs9.mag.dat infmt=takahasi
       options=stop lbox=0.2 o=takas_zs9


       The with the output file we run tpcf:


       time tpcf in=Input/takas_zs9.txt options=compute-HistN verb=2
       sizeHistN=40 rangeN=0.02 search=tree-omp



       time tpcf o=output options=compute-HistN verb=2 nbody=64000
       search=direct-omp-sincos


       Using python scripts you can generate several plots:


       cd python


       python plotting_points.py


       python plotting_histN_histXi2pcf.py


       python3 plotting_zetaM.py


       To smooth a data points (reducing its number) do:


       tpcf o=output ofmt=columns-ascii options=smooth verb=2 nbody=64000 nsmooth=4
